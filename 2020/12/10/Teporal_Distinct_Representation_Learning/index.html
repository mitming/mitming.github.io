Temporal Distinct Representation Learning for Action Recognition
Created2020-12-10|Updated2020-12-10|Video
|Word count:1.5k|Reading time:5min

论文: Temporal Distinct Representation Learning for Action Recognition

这篇论文是 AAAI 2020 TEINet:Towards an Efficient Architecture for Video Recognition 的 follow-up, 主要改进是让 Feature Selection Module 有全局的信息, 并加了个 loss 迫使网络学时序信息, 也是同一组人搞的.

Conclusion

由于一个 video 的不同帧共享一个 2D CNN 权重, 所以 2D CNN 会倾向于学习重复的, 冗余的外观信息, 从而忽视掉外观信息的变化差异, 最终导致一种伪过拟合(只记住了训练集的外观, 而非动作), 为了解决这个问题, 作者分 2 步走, 第一步先用 attention 来增强当前帧的某些通道(和动作相关), 抑制另一些通道(和动作无关), 第二步用余弦相似度衡量 2 帧在每个通道上特征图的相似度, 把这个余弦相似度加入到 loss 中迫使整个网络去学习时序信息(学外观差异而不是学相似的外观信息)

Idea

2D CNN 处理 video 的方式是把 (batch,time,c,h,w)→(batch∗time,c,h,w)(batch, time, c, h, w) \rightarrow (batch*time, c, h, w)(batch,time,c,h,w)→(batch∗time,c,h,w), 再进行卷积, 然后利用 TSM 模块或者时序卷积捕获时序信息, 这样做的坏处是同一个 video 的所有帧的外观信息是有很强的相似度的, 会让同一个 video 的所有帧共享 2D CNN 的权重, 这会导致 2D CNN 去学习帧和帧之间相似的外观信息了, 而不是去学外观差异导致的动作信息, 这就导致了冗余, 因此 2D CNN 的方法会学习更多的场景特征, 而忽略帧和帧之间的变化.

为了解决这个问题, 作者分为 2 步, 第一步先用 attention 来增强当前帧的一些通道, 抑制另一些通道, 然后在损失函数上添加帧和帧之间每个通道的余弦相似度, 迫使帧和帧的相同通道尽可能不同, 以这种形式来迫使网络学 motion 信息.

Method

先回顾下 TEI 里面的通道增强模块(MEM)和时序建模模块(TIM)

MEM 用相邻 2 帧特征图之差 + attention 来增强一些通道, 抑制另一些通道, TIM 模块本质就是时序卷积, 用来捕获时序信息.

这里对 MEM 模块进行了改进(下图中左半部分所示), MEM 只用 2 帧之差来做 attention, 这里用所有帧的信息来对当前帧的通道做 attention, 方法是在每个时间步都把 2 帧的差(下图中的d)给累加起来, 作为 memory(下图中的m)

对于一个分类问题, 其损失函数可以写成:

Experiments

之前有文章说了卷即网络的早期阶段更注重外观建模, 后面的 layers 更注重时序建模[1][2]^{[1][2]}[1][2], 所以只在网络每个 layer 的后半部分加 temporal diversity loss, 放的位置在 ResBlock 的 ResConv(1×1+3×3+1×1 conv1 \times 1 + 3 \times 3 + 1 \times 1 \space conv1×1+3×3+1×1 conv) 之前, 如图所示, 其中 TDRL-A 没有 temporal diversity loss, TDRL-B 有 temporal diversity loss, 只在每个 layer 的最后添加 temporal diversity loss

Results

最开始都是 2 个大表, 在 Kinetics 和 Something-Something 上的 SOTA 结果, 有点疑惑的是这里的 FLOPs 计算, 为什么 8f×28f \times 28f×2 比 16f16f16f 高了这么多?

然后是一些消融性验证, 下面左边的表的 backbone 是 ResNet-50 + TSM, 右边表的 backbone 是 ResNet-50 + TIM, TSM 和 TIM 都是用来建模时序信息的模块, 做这个实验的目的是证明 PEM 和 TD Loss 不管对于什么方法都应该是有效的

最后就是一些参数的设置实验, 从左到右分别是:

  1. 拿多少通道来计算 TD Loss
  2. 在模块的哪个位置插入 PEM 和 TD Loss
  3. 正则化项(TD Loss)的权重

这个结果表明不是所有的通道都需要加入到正则化项, 因为肯定不是所有的通道都对时序建模有帮助, 即使在前面做了 attention. 此外, TD Loss 放在时序建模模块之后才更有效果.
