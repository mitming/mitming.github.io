MotionSqueeze:Neural Motion Feature Learning for Video Understanding

论文: MotionSqueeze:Neural Motion Feature Learning for Video Understanding

Conclusion

解决的问题是如何在模型中学习 motion 特征, 而不用花费大量的算力和资源计算光流再用 two-stream 方法, 作者提出了一个模块, 把这个模块加到 TSM 中来实现 motion feature 的建模, 这个模块分 3 步, 首先计算相邻 2 帧之间的 local correlation, 然后将其转化为 motion 特征(主要是这一步), 最后将其和原来的输入特征图做特征融合.

Idea

通常的做法是学习一个类似 FCN 的结构来隐式学习 optical flow, 但这种结构比较耗时, 另一种做法是直接用帧和帧的特征图之差来代表 motion 信息, 这种方法虽然快但精度不行(论文是这样说的, 有所怀疑), 所以作者提出了个这么个方法, 能比较有效率地学习 motion 特征且达到和 external optical flow 的精度.

Method

如下图所示, 整个 MS Module 分 3 步, 先后分别是计算相关性, 位移估计和特征迁移.

Correlation Computation

对于前一帧特征图的第 xxx 个位置 Fx(t)F^{(t)}_xFx(t)​, 定义这个位置处在位移为 ppp 情况下的 relation score 为:

s(x,p,t)=Fx(t)⋅Fx+p(t+1)s(\mathbf{x}, \mathbf{p}, t)=\mathbf{F}_{\mathbf{x}}^{(t)} \cdot \mathbf{F}_{\mathbf{x}+\mathbf{p}}^{(t+1)} s(x,p,t)=Fx(t)​⋅Fx+p(t+1)​

其中 Fx(t)F^{(t)}_xFx(t)​ 的 size 为 (1, 1), Fx+p(t+1)\mathbf{F}_{\mathbf{x}+\mathbf{p}}^{(t+1)}Fx+p(t+1)​ 的 size 为 (P, P), 其中 p∈[−k,k]2,P=2k+1p \in [-k, k]^2, P=2k+1p∈[−k,k]2,P=2k+1, 由于第 ttt 帧的 spatial size 为 H×WH \times WH×W, 所以第 ttt 帧的 relation score 的 size 为 H×W×P2H \times W \times P^2H×W×P2, 这个计算量相当于 P2P^2P2 个 1×11 \times 11×1 卷积, 计算过程可以把 t+1t+1t+1 帧的特征图中以 xxx 位置为中心的 P2P^2P2 的特征图作为 kernel.

总的 FLOPs 相当于 THWCP2THWCP^2THWCP2, 在实际使用时, 只把这个模块放到 ResNet 的第 3 个 stage 中, 所以其特征图 spatial size 为 28×2828 \times 2828×28, 并设置 P=15P=15P=15, 在计算相关性之前先用一个 1×11 \times 11×1 卷积把通道数减半.

Displacement Estimation

从上一步得到了 relation tensor S(t)S^{(t)}S(t), 这一步通过这个为 motion 计算位移场. 为了从下一帧中找到当前帧位置 xxx 处有最大 relation score 的位置 ppp, 最简单但却不可微的方法是计算 argmax⁡ps(x,p,t)\operatorname{argmax}_{\mathbf{p}} s(\mathbf{x}, \mathbf{p}, t)argmaxp​s(x,p,t), 即求出 sss 具有最大响应值时的 ppp, 为了使这个操作可微, 作者使用 softmax 来得到位移的加权平均值:

d(x,t)=∑pexp⁡(s(x,p,t))∑p′exp⁡(s(x,p′,t))pd(\mathbf{x}, t)=\sum_{\mathbf{p}} \frac{\exp (s(\mathbf{x}, \mathbf{p}, t))}{\sum_{\mathbf{p}^{\prime}} \exp \left(s\left(\mathbf{x}, \mathbf{p}^{\prime}, t\right)\right)} \mathbf{p} d(x,t)=p∑​∑p′​exp(s(x,p′,t))exp(s(x,p,t))​p

但这种方法受所有相关值的影响, 因此对 correlation tensor 中的噪声(离群值)很敏感, 所以利用 2D 高斯核来抑制噪声:

d(x,t)=∑pexp⁡(g(x,p,t)s(x,p,t)/τ)∑p′exp⁡(g(x,p′,t)s(x,p′,t)/τ)pd(\mathbf{x}, t)=\sum_{\mathbf{p}} \frac{\exp (g(\mathbf{x}, \mathbf{p}, t) s(\mathbf{x}, \mathbf{p}, t) / \tau)}{\sum_{\mathbf{p}^{\prime}} \exp \left(g\left(\mathbf{x}, \mathbf{p}^{\prime}, t\right) s\left(\mathbf{x}, \mathbf{p}^{\prime}, t\right) / \tau\right)} \mathbf{p} d(x,t)=p∑​∑p′​exp(g(x,p′,t)s(x,p′,t)/τ)exp(g(x,p,t)s(x,p,t)/τ)​p

其中 ggg 是高斯核, 其标准差设置为 5, τ\tauτ 是用来调节 softmax 分布的因子, 随着 τ\tauτ 的减少, softmax 会逐渐接近 argmax, 文中设置为 0.01

g(x,p,t)=12πσexp⁡(p−argmax⁡ps(x,p,t)σ2)g(\mathbf{x}, \mathbf{p}, t)=\frac{1}{\sqrt{2 \pi} \sigma} \exp \left(\frac{\mathbf{p}-\operatorname{argmax}_{\mathbf{p}} s(\mathbf{x}, \mathbf{p}, t)}{\sigma^{2}}\right) g(x,p,t)=2π​σ1​exp(σ2p−argmaxp​s(x,p,t)​)

除了使用上面的 estimated displacement map 外, 作者还使用了 confidence map 作为额外的 motion 信息, 通过在每个位置 xxx 处 pooling 最高的相关性得到:

s∗(x,t)=max⁡ps(x,p,t)s^{*}(\mathbf{x}, t)=\max _{\mathbf{p}} s(\mathbf{x}, \mathbf{p}, t) s∗(x,t)=pmax​s(x,p,t)

其中 displacement map 有 2 个 channel, confidence map 有 1 个 channel, 作者将它们 concatenate 起来, 所以上图中 Displacement Estimation 的输出为 H×W×3H \times W \times 3H×W×3

Feature Transform

这一部分的目的是把 Displacement Estimation 得到的特征图的 size 从 H×W×3H \times W \times 3H×W×3 变为 H×W×CH \times W \times CH×W×C, 以便和 identity 做 element-wise add, 即特征融合操作.

这里作者简单地用了 4 个 DW 卷积, size 分别是一个 1×7×71 \times 7 \times 71×7×7 和后面 3 个 1×3×31 \times 3 \times 31×3×3, 感觉这一步有点没道理, 明明一个 1×11 \times 11×1 卷积就能做的事情, 为什么要用 4 个 DW 卷积来完成? 感觉纯粹是试出来的?

Results

国际惯例, 先看在 something-something 上的结果.

然后作者比较了自己的方法和其他 motion representation 方法在 kinetics 和 HMDB 51 上的结果??? kinetics 和 HMDB51 都是更看重外观信息吧, 为什么你要在这上面做实验来对比和其他 motion 表征方法的结果???

作者的解释居然是之前的 motion representation 方法通常在这 2 个数据集上汇报结果 ??? what the fuck ???

接下来是消融性验证

baseline 就是 TSM-R50, S 代表使用 softmax 来计算 displacements 的加权平均, KS 代表使用高斯核来抑制噪声, CM 代表 confidence map, BD 代表用 F(t)F^{(t)}F(t) 来得到 F(t+1)F^{(t+1)}F(t+1) 的 correlation score, 然后和 F(t+1)F^{(t+1)}F(t+1) 计算得到的 F(t)F^{(t)}F(t) 的 correlation score 给 concatenate 起来.

在 something-something v1 数据集上的结果上, 加了 MSModule 后直接涨了 2 个多点, 如果使用高斯核来降噪, 还能再涨一个点, CM 和 BD 也能带来一点精度增长.

但这里作者并没有做 feature transform 的消融性验证, 没有说明为什么要用 4 个 DW 卷积 ???

然后是不同的 patch size(P 的大小) 和 FLOPs 的 trade-off:

, 根据作者选择了 P=15, 此时有最佳的 trade-off.

然后是把 MS Module 放在不同位置的精度, 以及不同特征融合方式的精度差异.

左图得到的结论是放在 res3 处时有更好的结果和 FLOPs, 右图的结论是只是用 MS Module 的输出特征图, 其精度不如 identity, 如果把它和 identity 做了特征融合, 那么精度会上升比较多.

最后为了证明这种方法的通用性, 在不同的 backbone 上做了实验, 也对比了双流网络的结果

从左图可以看出 MS Module 主要是对没有时序建模能力的 backbone 提升比较大, 对有时序建模能力的 backbone 提升稍微低了点, 然后就是对 2D CNN 方法的提升会比 3D CNN 的提升来的大.

右图是和 TVL1 光流的结果, Nr,NfN_{r}, N_{f}Nr​,Nf​ and NsN_{s}Ns​ 分别表示帧的数目, 光流的数目以及 stacking size 的大小.

最后作者还把 displacement map 和 confidence map 可视化了一下, 表明 MS Module 的作用确实类似于光流, 用于表征 motion.
