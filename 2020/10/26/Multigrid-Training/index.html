Multigrid Training
Created2020-10-26|Updated2020-10-27|Video
|Word count:2.9k|Reading time:10min

论文地址
开源代码

目前视频模型的训练通常是 follow 图像模型的训练过程, 即有着固定的 mini-batch data shape: 特定数目的 clips, frames 以及空间大小

但这种 shape 是最优的吗? 通常来说高分辨率(时间和空间维度上)的模型训练慢, 但精度高, 低分辨率的模型训练快但精度低, 收到数值优化中 multigrid 方法的启发, 本文提出使用可变的 mini-batch shapes, 即根据一种采样策略来在训练时改变 mini-batch 中的时空分辨率, 也就是改变 mini-batch 中数据的 shape, 在减少时间和空间维度的同时改变 mini-batch 的大小以及学习率大小, 以这种方式来加速训练.

Introduction

对于 3D CNN 来说, 训练时优化器每次迭代会处理一个 mini-batch 的数据, 这个 mini-batch 里面的数据的形状 B×T×H×WB \times T \times H \times WB×T×H×W 在整个训练过程中都是固定的(CCC 忽略不用管), 有一些方法试图通过增大这些维度来得到更好的精度, 本文的思考的问题是, 能否通过在训练过程中改变 mini-batch 中数据的 shape 来在不减少精度的同时加快模型训练?

通常来说在更好的精度和更快的训练之间是存在一个 trade-off 的, 如果使用更少时空分辨率的数据, 那么就可以增大 mini-batch size BBB, 这样就能用更少的时钟时间来处理相同数量的 epoch, 因为每次迭代都能处理更多的样本. 这篇文章的中心思想就是避免这种 trade-off, 即通过在训练中不断改变 mini-batch 的 shape, 达到在不掉精度的情况下加快网络训练的效果.

可以把一个 mini-batch 中输入的 video clips 看成是 raw video 在一个 sampling grid 上的采样, 这样就能够使用数值分析中的 multigrid 方法来处理. 这种方法是使用从粗到细的 grids 来加速优化的.

直观的说, 如果在训练的早期使用 large mini-batch size 和 small time and space dimensions, 即 coarse grid, 在训练的后期使用 small mini-batch size 和 large time and space dimensions, 即 fine grid, 优化器(如 SGD) 可能能够更快地扫描完所有数据(我的理解是能更快地帮助模型收敛).

看下 multigrid training 的效果, 一图胜千言.

Concepts

Sampling Grids

一个维度上的 sampling grid 由 2 个属性决定: span 和 stride, 它们的基本单元分别是 frames 和 pixels. span 定义了整个 sampling grid 能够涵盖的范围, stride 是采样点之间的距离. 有了 stride 和 span 后就能得到一个 grid 中采样点的数目, 也就决定了输入数据的 shape. 我自己也没怎么看明白这段话, 后面做了个图来理解什么是 sampling grid, 如下所示

上图中绿色的为 raw video 的一个 clip, 这个 clip 并非是模型的真正输入, 通常还需要对这个 clip 进行采样, 采样的结果为橙色的部分, 橙色的部分才是模型的真正输入, 其形状就是后面所说的 data shape.

首先定义正常训练的 data shape 为 B,T,H,WB, T, H, WB,T,H,W, 使用 multigrid training 的 data shape 为 b,t,h,wb, t, h, wb,t,h,w.

通常来说一个 clip 的 size 为 64 帧, 在保持原始纵横比不变的情况下, 长和宽比较小的那个有 256 个像素点, 这里假设长为256, 宽为512. 传统训练方法是怎么做的呢? 通常会按照 8×88 \times 88×8 来进行采样, 然后 crop 一个 256×256256 \times 256256×256 的 image 来进行训练, 即 T=8,W=H=256T=8, W=H=256T=8,W=H=256.

multigrid training 的目的是通过减小 T,H,WT, H, WT,H,W 的值来加速计算, 为了保持 FLOPs 大致不变, 就需要改变此时 batch size 的大小:

b=BTtHhWw(1)\begin{aligned} &b=B \frac{T}{t} \frac{H}{h} \frac{W}{w}\\ \end{aligned} \tag{1} ​b=BtT​hH​wW​​(1)

作者提到这里的 spatial sampling grids 其实就是 multi-scale spatial 训练, 在本文的观点下, 多尺度训练是成比例地改变了 w,hw, hw,h, 但本文的方法是改变 t,h,wt, h, wt,h,w, 这样对于不同的 grid 会有不同的时间和空间维度, 所以多尺度训练可以看成是 multigrid training 的一种特例.

Grid Scheduling

上面解释了 sampling grid 的概念, 对于 multigrid training 而言有一组 sampling grids 和一个 grid schedule 来决定在每次 iteration 中使用哪个 sampling grid. 如果不考虑 grid 的选择, 只考虑训练相似数量的 epoch 的话, 在整个训练过程中只要使 b>Bb > Bb>B 就能减少总的 FLOPs 和时钟时间.(之前有说明)

基于此作者引出了 2 个问题:

  1. 是否存在一个 grid 的集合, 使得在更快训练的同时保持精度?
  2. 如果存在, 是否可以不加修改地推广到新的模型和数据集?

Multigrid Properties

Multigrid training 的 2 个属性是数据和模型, 首先如果要在不同的 grids 上对数据进行 resampling 需要一个合适的 operator. 对于 video 来说, 这个 operator 可以是用于 source discrete signal 的 reconstruction filter, 这个点上的值可以利用 grid 计算出来(例如双线性插值). 其次对于模型而言, 如果需要在不同的 grids 上进行 resampling, 那么就需要对在不同 grids 上采样的输入兼容.

所有的模型都以 GAP 再加全连接层作为分类器.

Training and Testing Distributions

这种训练方式会导致训练时的数据和测试时的数据分布不一致, 这个问题最开始在 Cascade R-CNN 就被提出, 为了解决这个问题, 在训练结束后使用和 test data 分布更对齐的 grid 再进行几次 fine-tuning, 作者发现这种 fine-tuning 能够带来一个比较小但却非常稳定的提升.

Implementation Details

Multigrid training 涉及了 sampling grids 及 grid schedule 的选择, 作者提出的分层的 schedule，以两种不同的频率在 mini-batch shapes 之间进行交替:

  1. long cycle
    在一组 base shapes 上进行移动, 在每个 shape 上持续几个 epochs
  2. short cycle
    在一组接近当前 base shapes 的 shapes 上移动, 在每个 shape 上持续一个 iteration

如下图所示:

Long Cycle

Long Cycle 有 4 组不同的 grids(即 num of shape S=4S=4S=4), 对应不同的 batch size, 在每一个 cycle 中每个维度都是非递减的, 这和前面说到的在训练的早期使用 large mini-batch size 和 small time and space dimensions, 即 coarse grid, 在训练的后期使用 small mini-batch size 和 large time and space dimensions, 即 fine grid 是一致的.

  • 8B×T4×H2×W28B \times \frac{T}{4} \times \frac{H}{\sqrt{2}} \times \frac{W}{\sqrt{2}}8B×4T​×2​H​×2​W​
  • 4B×T2×H2×W24B \times \frac{T}{2} \times \frac{H}{\sqrt{2}} \times \frac{W}{\sqrt{2}}4B×2T​×2​H​×2​W​
  • 2B×T2×H×W2B \times \frac{T}{2} \times H \times W2B×2T​×H×W
  • B×T×H×WB \times T \times H \times WB×T×H×W

long cycle 和 step-wise 学习率衰减策略同步, 即在固定的 step 时的学习率是一致的, 对于每个阶段的学习率, 都恰好有一次完整的 cycle 发生, 每次 cycle 都会处理 SSS 个shape.

使用一种简单的随机策略来为每次 training iteration 生成一个特定 shape 的 mini-batch: 对于 mini-batch 中的每个 video, 从特定的 span 中选择一个随机的 span 并设定一个 stride, 以便在 resulting grid 上采样时有想要的 shape.

对于空间维度, 这种策略等于使用双线性插值来 resize 一个 random crop 到想要的 shape. 对于时间维度, 这种策略相当于选择一个随机的 temporal crop 并对它的 frames 进行下采样.

Short Cycle

short cycle 在一个可变的 spatial shape 上快速移动, 不在 temporal shape 上移动, 所以这里的 TTT 是固定的, 这里使用 3-shape 的 short cycle, 对于每次迭代 iii, m=(imod  3)m = (i \mod 3)m=(imod3)

  • m=0m = 0m=0, 设置 spatial shape 为 H2×W2\frac{H}{2} \times \frac{W}{2}2H​×2W​
  • m=1m = 1m=1, 设置 spatial shape 为 H2×W2\frac{H}{\sqrt{2}} \times \frac{W}{\sqrt{2}}2​H​×2​W​
  • m=2m = 2m=2, 设置 spatial shape 为 long cycle 中的 base spatial shape H,WH, WH,W

short cycle 可以和 long cycle 结合起来用.

Learning Rate Scaling

随着 batch size 的改变, 使用 linear scaling rule 来调整学习率, batch size 增大多少倍, 学习率就增大多少倍. 作者通过实验发现在 short cycle 时这样做会掉精度, 所以只在 long cycle 阶段对学习率进行调整.

Fine-tuning Phase

假设不使用 multigrid training 的方法有 LLL 个学习率 stages, 作者在前 L−1L-1L−1 个学习率阶段中使用 long cycles 和 short cycles. 在第 LLL 个学习率阶段来做 fine-tuning 来对训练分布和测试分布进行 match.

在 fine-tuning iterations 的前半段使用第 L−1L-1L−1 阶段的学习率, 在后半段使用第 LLL 阶段的学习率, 在 fine-tuning 时使用 short cycle 而不使用 long cycle.

Results

首先看下总的结果

图中的标注 1.5×1.5\times1.5× 是指相比于默认的 baseline 训练数量(约 112k iterations 或者 239 个epochs), 多训练了 1.5 倍的 epochs 数量.

可以看到如果只是增大 batch size 并减小 data shape, 效果并不好, 比 baseline 都低了很多(紫线), baseline 是蓝线.

黑线和红线分别代表 long cycle 和 short cycle, 可以看到在训练相同的 epochs/iterations 情况下不但训练时间减少了很多, 而且精度还不低, 同时使用 long cycle 和 short cycle 的话效果更是爆炸, 在精度相近且都训练 112k iterations 的情况下, multigrid training 所需要的时钟时间仅为 3 小时, 而 baseline 训练所需要的时钟时间为 20 多个小时…

接下来做了消融性验证, 证明了 long cycle 在 4-shape 的情况下最优, short cycle 在 3-shape 的情况下最优

接下来是泛化性分析, 证明了 multigrid training 不管是对于预训练还是从头训练的模型都有很好的效果, 对于不同的 temporal shape 和 spatial shape 的模型也都有很好的效果.

Conclusion

我个人感觉有点类似于多尺度训练, 和传统多尺度训练方法不同的是这里为 video 数据添加了时间维度上的多尺度, 同时为了加快训练速度和提高精度, 采用了 long-cycle, short-cycle 这 2 种多尺度的训练策略. 这篇和 SlowFast 都可以看做是把传统方法拿到 video 上, 并在此基础上针对 video 数据的 temporal 维度做了很多优化.
