RubiksNet:Learnable 3D-Shift for Efficient Video Action Recognition
Created2020-10-30|Updated2020-10-30|Video
|Word count:3k|Reading time:12min

这篇文章算是 TSM 的 follow up, TSM 那篇文章中 shift 哪些通道是自己手工设定的, 这篇文章想让网络自己去学习 shift 哪些通道, 另外 TSM 只在 temporal 维度上 shift channels, 而这里在 T,H,WT, H, WT,H,W 上都 shift channels.

Introduction

虽然 TSM 通过在时序维度上 shift channels 来减少计算量, 但作者认为 TSM 模块的效率仍然会受限于空间操作中的参数和计算量, 所以想在 HHH 和 WWW 维度也对 channel 进行 shift, 即这篇文章不但把 shift channels 的操作拓展到了空间上(H and W), 而且还把 hand-designed fixed temporal shift 变成了 learnable unfixed spatiotemporal shift, 可以从名字上直观地看出把 hand-designed →\rightarrow→ learnable, fixed →\rightarrow→ unfixed, temporal →\rightarrow→ spatiotemporal.

视频序列中连续帧的时空上下文信息存在冗余的情况, 这种冗余信息会影响 action recognition 的效果, 所以需要同时考虑在空间和时序维度上的 shift 操作, 并且这个操作不能像 TSM 一样是固定分配的(TSM 中每次都 shift 1/81/81/8 的通道). 但由于 design space 太难以解决, 所以无法简单地将 fixed temporal shift 泛化为 unfixed spatiotemporal shift. 所以我们需要研究每个维度上的哪些 channels 能够被 shift, 能被 shift 的幅度有多少以及结合了这些操作的基本设计及其他设计选择. 所以这里自然而然地就有了一个 motivation, 那就是让模型自己去学这些 shift 操作.

鉴于此作者提出了 RubiskNet, 通过 spatiotemporal shift layer 来同时在空间和时序上下文学习如何进行 shift 操作, 作者探索了这种 design 的几个变种, 发现可以通过添加一个 constrained shift budget 来学习对 shift 操作更加有效的 allocation.

RubiksNet 和 TSM 的区别如下图所示, TSM 是 fixed temporal shift, fixed 是指有着固定的 shift allocation(每次都 shift 1/81/81/8 的通道), 而 RubiksNet 是 learned spatiotemporal shift, 其 allocation 在 shift budget constraint 下学习到的.

P.S. 作者还总结了最近 action recognition 主要研究的 2 个方向:

  1. 通过混合 3D 卷积和 separate spatial + temporal 卷积来提升效率.
  2. 添加 long term temporal reasoning 来更好地提升方法的精度.

Learnable 3D Shift

Spatiotemporal Convolution

对于特征 F∈RC×T×H×WF \in \mathbb{R}^{C \times T \times H \times W}F∈RC×T×H×W, 使用 3D convolution K∈RC′×C×TK×HK×WKK \in \mathbb{R}^{C^{\prime} \times C \times T_{K} \times H_{K} \times W_{K}}K∈RC′×C×TK​×HK​×WK​, 输出的 tensor 为 O∈RC′×T×H×WO \in \mathcal{R}^{C^{\prime} \times T \times H \times W}O∈RC′×T×H×W, 那么 3D 时空卷积的操作可以被定义为:

Oc′,t,h,w=∑c,i,j,kKc′,c,k,i,jFc,t+k^,h+i^,w+j^(1)O_{c^{\prime}, t, h, w}=\sum_{c, i, j, k} K_{c^{\prime}, c, k, i, j} F_{c, t+\hat{k}, h+\hat{i}, w+\hat{j}} \tag{1} Oc′,t,h,w​=c,i,j,k∑​Kc′,c,k,i,j​Fc,t+k^,h+i^,w+j^​​(1)

其中 i,j,ki, j, ki,j,k 分别为卷积核中沿 temporal, height 和 width 维度的索引, c,c′c, c'c,c′ 为 channel 维度的索引, 索引 i^,j^,k^\hat{i}, \hat{j}, \hat{k}i^,j^​,k^ 是重新中心化(re-centered)的空间和时序索引, 其中 k^=k−⌊TK/2⌋,i^=i−⌊HK/2⌋,j^=j−⌊WK/2⌋\hat{k} = k-\left\lfloor T_{K} / 2\right\rfloor, \hat{i} = i-\left\lfloor H_{K} / 2\right\rfloor, \hat{j}=j-\left\lfloor W_{K} / 2\right\rfloork^=k−⌊TK​/2⌋,i^=i−⌊HK​/2⌋,j^​=j−⌊WK​/2⌋

为了方便假设 H=WH=WH=W, 那么 3D 卷积总的参数量为 C×C′×TK×HK2C \times C' \times T_{K} \times H_{K}^{2}C×C′×TK​×HK2​, 总的计算量为 C×C′×(T×H2)×(TK×HK2)C \times C^{\prime} \times\left(T \times H^{2}\right) \times\left(T_{K} \times H_{K}^{2}\right)C×C′×(T×H2)×(TK​×HK2​)

Fixed Spatiotemporal Shift

这里作者提出使用一个时空 shift 操作来代替传统的 3D 卷积. 由于 shift 操作可以看成是一种特殊的深度可分离卷积, 所以可以把时空 shift 操作表示为:

Oc,t,h,w′=∑i,j,kSc,k,i,jFc,t+k^,h+i^,w+j^(2)O_{c, t, h, w}^{\prime}=\sum_{i, j, k} S_{c, k, i, j} F_{c, t+\hat{k}, h+\hat{i}, w+\hat{j}} \tag{2} Oc,t,h,w′​=i,j,k∑​Sc,k,i,j​Fc,t+k^,h+i^,w+j^​​(2)

其中 S∈{0,1}C×TK×HK×WKS \in\{0,1\}^{C \times T_{K} \times H_{K} \times W_{K}}S∈{0,1}C×TK​×HK​×WK​, 当且仅当 (i,j,k)=(ic,jc,kc)(i, j, k) = (i_c, j_c, k_c)(i,j,k)=(ic​,jc​,kc​) 时 Sc,k,i,j=1S_{c, k, i, j} = 1Sc,k,i,j​=1, 其中 ic,jc,kci_c, j_c, k_cic​,jc​,kc​ 是和通道下标 ccc 对应的 spatiotemporal shift. 在实际操作中, 能够索引到适当的存储器地址来高效地实现这种操作, 即这种 shift operation 无需任何浮点操作.

接着对公式(2)的输出使用一个 pointwise 卷积来整合跨通道的通道, 以此来得到最终的输出:

Oc′,t,h,w=∑cPc′,cOc,t,h,w′(3)O_{c^{\prime}, t, h, w}=\sum_{c} P_{c^{\prime}, c} O_{c, t, h, w}^{\prime} \tag{3} Oc′,t,h,w​=c∑​Pc′,c​Oc,t,h,w′​(3)

同样假设 H=WH=WH=W, 那么这种操作总的计算量为 C×C′×(T×H2)C \times C' \times (T \times H^2)C×C′×(T×H2), 远低于 Spatiotemporal Convolution 的 C×C′×(T×H2)×(TK×HK2)C \times C^{\prime} \times\left(T \times H^{2}\right) \times\left(T_{K} \times H_{K}^{2}\right)C×C′×(T×H2)×(TK​×HK2​)

Learnable Spatiotemporal Shift

首先考虑把传统的离散 shift 操作变为连续操作, 这样才能直接使用反向传播来优化 shift 参数, 定义可以学习的 3D shift 参数为:

θ={(γc,αc,βc)∣c∈C}(4)\theta=\left\{\left(\gamma_{c}, \alpha_{c}, \beta_{c}\right) \mid c \in C\right\} \tag{4} θ={(γc​,αc​,βc​)∣c∈C}(4)

其中 γc,αc,βc\gamma_{c}, \alpha_{c}, \beta_{c}γc​,αc​,βc​ 分别是对每个通道 ccc 的时序, 垂直和水平维度的 shift 参数, 然后考虑方程的另一种表达方式:

Oc,t,h,w′=∑i,j,kSc,k,i,jθFc,t+k^,h+i^,w+j(5)O_{c, t, h, w}^{\prime}=\sum_{i, j, k} S_{c, k, i, j}^{\theta} F_{c, t+\hat{k}, h+\hat{i}, w+j} \tag{5} Oc,t,h,w′​=i,j,k∑​Sc,k,i,jθ​Fc,t+k^,h+i^,w+j​(5)

Sc,k,i,jθ=∏(z,g)∈{(γc,k),(αc,i),(βc,j)}{Δz if g=⌈z⌉1−Δz if g=⌊z⌋0 otherwise (6)S_{c, k, i, j}^{\theta}=\prod_{(z, g) \in\left\{\left(\gamma_{c}, k\right),\left(\alpha_{c}, i\right),\left(\beta_{c}, j\right)\right\}}\left\{\begin{array}{ll} \Delta z & \text { if } g=\lceil z\rceil \\ 1-\Delta z & \text { if } g=\lfloor z\rfloor \\ 0 & \text { otherwise } \end{array}\right. \tag{6} Sc,k,i,jθ​=(z,g)∈{(γc​,k),(αc​,i),(βc​,j)}∏​⎩⎨⎧​Δz1−Δz0​ if g=⌈z⌉ if g=⌊z⌋ otherwise ​(6)

Δz=z−⌊z⌋\Delta z = z - \lfloor z \rfloorΔz=z−⌊z⌋, ggg 是对应于参数维度 zzz 的下标(例如 γc\gamma_cγc​ 对应于时间下标 kkk), 这里的每个稀疏项 Sc,k,i,jθS_{c, k, i, j}^{\theta}Sc,k,i,jθ​ 都是一个系数, 用来表示来自所有 3 个维度的 interpolated shift contributions 的乘积.

直观的来看, 这里做的事情是在一个显式的连续的 trilinearly-interpolated input activation tensor 上进行 shift 操作. 在每个 shift 参数的位置, 围绕每个 shift 运算在其局部领域(232^323-point cube)周围稀疏地评估 interpolation

作者表明这个方程仍然只是形式主义, 实际上能够以很小的额外开销把操作写入 GPU 上, 作者也在补充材料里面提供了额外的技术讨论. 这样就得到了最终的 learnable spatiotemporal shift operation RubiksShift

Interpolated Quantized Shift

尽管上面的插值公式已经很有效了, 但还可以通过量化来进一步提升效率, 作者提的第一个 RubiksShift 的变体是基于 interpolated shift quantization 机制的简单时空拓展.

在训练时, 模型将插值后的 shift 参数的 copy 保持为浮点数, 在 forward 阶段, 所有的 shifts 都被舍入到最接近的整数, 在 backward 阶段, 计算连续 shifts 的梯度, 这些参数通过梯度下降来更新, 作者做了消融性验证, 表明这种方法虽然确实能 work, 但缺乏真实的梯度会最终极大地阻碍精度. (我的理解是用整数的梯度来替代浮点数的梯度???)

Attention Quantized Shift

为了解决插值量化变体带来的问题, 作者进一步提出了 RubiksShift-AQ, 如下图所示

作者把这种方式和 2D CNN 中的 attention 剪枝操作联系起来, 根据 integer shift values 对应的 attention 分布来对参数进行 shift. 为了在 inference 时也能有这种高效率, 对于给定的 attention weight tensor WattnW_{attn}Wattn​, 计算:

Wattnshift =softmax⁡(τaWattn std(Wattn ))(7)\mathbf{W}_{\text {attnshift }}=\operatorname{softmax}\left(\tau_{a} \frac{\mathbf{W}_{\text {attn }}}{\text {std}\left(\mathbf{W}_{\text {attn }}\right)}\right) \tag{7} Wattnshift ​=softmax(τa​std(Wattn ​)Wattn ​​)(7)

τa\tau_{a}τa​ 是 softmax 的温度, 在 softmax 操作之后, WWW 的每个值都被正则化为 0 到 1 中的某个数, 代表注意力权重. τa\tau_{a}τa​ 高的时候, WWW 中的值就更接近均匀, 在 τa\tau_{a}τa​ 很低的时候, WWW 就是一个 binary tensor, 每个 channel 中只有一个非 0 项, 在训练时将 τa\tau_{a}τa​ 指数地减少到一个很低的值, 通常取 10−310^{-3}10−3

然后使用 hard max operation 把 attention shift WattnshiftW_{attnshift}Wattnshift​ 转化为等效的离散整数 shift 操作 SSS

Shift Operations Budget Constraint

TSM 已经提到了尽管 shift 操作不会带来参数量和 FLOPs 的增长, 但仍然会带来 latency cost, 鉴于此作者添加了一个 constraint, 用来惩罚模型的过度 shift. Budget constraint loss 如下所示:

Lbudget=∥1NL∑l=1NL(1C′∑c,∗Wnonzero (l))−B∣∣(8)L_{b u d g e t}=\| \frac{1}{N_{L}} \sum_{l=1}^{N_{L}}\left(\frac{1}{C^{\prime}} \sum_{c, *} \mathbf{W}_{\text {nonzero }}^{(l)}\right)-B|| \tag{8} Lbudget​=∥NL​1​l=1∑NL​​(C′1​c,∗∑​Wnonzero (l)​)−B∣∣(8)

其中 NLN_LNL​ 是 layers 的数量, BBB 是 0 到 1 之间的 shift budget 值, Wnonzero (l)\mathbf{W}_{\text {nonzero }}^{(l)}Wnonzero (l)​ 是在 lll 层的非 0 shift 位置的 attention 权重.

Model Architecture Design

Stable Shift Training

为了更稳定地进行 spatiotemporal shift 训练, 作者 normalize 了 shift gradients, 即只使用方向的改变, 而不是幅度.

实际上对 3 个维度都进行同等的 normalization 并不是最优的, 因为空间维度(224 * 224)比时序维度(8)要大的多, 所以作者提出使用 scaled gradient normalization, 把空间 shifts Δθh,Δθw\Delta \theta_{h}, \Delta \theta_{w}Δθh​,Δθw​ 和时序 shift \Delta \theta_{t} 的梯度向量缩放到一个椭圆体上而非单位球体上:

Δθh‾=ΔθhZ;Δθw‾=ΔθwZ;Δθt‾=λΔθtZ(9)\overline{\Delta \theta_{h}}=\frac{\Delta \theta_{h}}{Z} ; \quad \overline{\Delta \theta_{w}}=\frac{\Delta \theta_{w}}{Z} ; \quad \overline{\Delta \theta_{t}}=\frac{\lambda \Delta \theta_{t}}{Z} \tag{9} Δθh​​=ZΔθh​​;Δθw​​=ZΔθw​​;Δθt​​=ZλΔθt​​(9)

其中 λ\lambdaλ 是时序梯度缩放因子, ZZZ 是正则化因子:

Z=∥Δθh∥2+∥Δθw∥2+λ∥Δθt∥2(10)Z=\sqrt{\left\|\Delta \theta_{h}\right\|^{2}+\left\|\Delta \theta_{w}\right\|^{2}+\lambda\left\|\Delta \theta_{t}\right\|^{2}} \tag{10} Z=∥Δθh​∥2+∥Δθw​∥2+λ∥Δθt​∥2​(10)
