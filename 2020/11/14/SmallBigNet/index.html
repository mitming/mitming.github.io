SmallBigNet
Created2020-11-14|Updated2020-11-17|Video
|Word count:2.5k|Reading time:10min

通常的时序卷积只作用于一个固定的视角, 这会导致时序卷积捕获 motion-unrelated 的上下文信息, 为了解决这个问题, 作者提出了 SmallBigNet, small 分支用来学习核心的语义, big branch 由于有更大的感受野, 所以能为 small branch 提供更加有效的特征. SmallBig 通过 local-to-global 的方式来建立时空上的联系, 而不是像 non-local 一样直接以 global 的方式建立联系.

上图的左边为一个标准的时序卷积(3×1×13 \times 1 \times 13×1×1), 这个黄色的 tube 用来识别 跳高 这个动作, 对于时刻 ttt 的 blue box 而言, 使用时序卷积带来的 t−1t-1t−1, t+1t+1t+1 的 yellow box 会提供无用甚至有害的上下文信息. 以这个为例, t−1t-1t−1 的 yellow box 包含了运动员的手臂, 但手臂和 跳高 这个动作很显然是没有什么关联的, 所以这就是一种冗余信息, 而 t+1t+1t+1 的 yellow box 则包括了运动员的上半身, 显然也是无用的, 所以简单地利用时序卷积回导致学到的特征不稳定, 甚至学到对结果不好的特征.

为了解决这个问题作者提出在传统 3D 卷积的基础上再加一个 big branch, vanilla 3D 卷积看成是 small branch, 在 big branch 上使用一个 max pool 来得到最能增强 small branch view 的上下文信息, 再利用权重共享的 1×1×11 \times 1 \times 11×1×1 卷积进行特征融合, 如上图中右半部分所示.

对于 small branch 中的 blue box, 找到其对应的 3×3×33 \times 3 \times 33×3×3 的 yellow tube, 利用 3D max pooling 找到响应值最大的特征, 即上图中的 red box, 为运动员的腿部(很显然腿部对于 跳高 这个动作是很有用的), 再和 small branch 中的特征进行融合.

最后 SmallBigNet 可以写成 ResNet-3D style 的方式, 如下图所示

  1. (a) 是 2D ResNet 结构
  2. (b) 是 3D ResNet 结构
  3. © 是在 3D ResNet 结构的基础上添加时序上的 SmallBig block (3×1×13 \times 1 \times 13×1×1)
  4. (d) 在 3D ResNet 结构的基础上添加时空上的 SmallBig block (3×3×33 \times 3 \times 33×3×3)
  5. (e) 是一个典型的 SmallBig Block, 对 3D ResNet 结构的每个 Conv 都添加 big branch.
  6. (f) 对 typical SmallBig block 的拓展, 其 big branch 作用于所有帧的所有空间上.

从结构图可以看出这里的 SmallBig block 是逐渐对全局信息进行建模的, 先建模 local relation, 再逐渐放大到 global relation. 以上图中的 (f) 为例, 最开始 2 个 3×3×33 \times 3 \times 33×3×3 Max Pool 用于对时空上的 size 为 3 的 tube 建模, 再用 T×3×3T \times 3 \times 3T×3×3 对所有帧在 3×33 \times 33×3 的空间上建模, 再到最后用 T×H×WT \times H \times WT×H×W 对所有帧的所有部分建模, 其时空感受野也是逐渐变大的.

可以看出这种方式和 non-local 是非常相似的, 区别在于 non-local 是直接对全局建模, SmallBig block 是先对 local 建模, 再对 global 建模, 作者认为这种 local-to-global 的方式更加有效, 还从数学上推导了一下, 推出 non-local 和 SmallBig block 本质上做的事情都是一样的: 对时空上下文进行建模.

先看下时空卷积的数学表达:

yt(h,w)=SmallBig⁡(Ψ,xt(h,w),{xk(i,j)})=Ψρxt(h,w)⏟small view+ΨνMaxPool⁡({xk(i,j)})⏟big view\begin{aligned} \mathbf{y}_{t}^{(h, w)} &=\operatorname{SmallBig}\left(\Psi, \mathbf{x}_{t}^{(h, w)},\left\{\mathbf{x}_{k}^{(i, j)}\right\}\right) \\ &=\underbrace{\Psi_{\rho} \mathbf{x}_{t}^{(h, w)}}_{\text {small view}}+\underbrace{\Psi_{\nu} \operatorname{MaxPool}\left(\left\{\mathbf{x}_{k}^{(i, j)}\right\}\right)}_{\text {big view}} \end{aligned} yt(h,w)​​=SmallBig(Ψ,xt(h,w)​,{xk(i,j)​})=small viewΨρ​xt(h,w)​​​+big viewΨν​MaxPool({xk(i,j)​})​​​

再看下 non-local 的数学表达, 为了方便先定义 {xk(i,j)}all\left\{\mathbf{x}_{k}^{(i, j)}\right\}_{a l l}{xk(i,j)​}all​ 为 global tube T×H×WT \times H \times WT×H×W 中的所有特征向量, 对于 (t,h,w)(t, h, w)(t,h,w) 处的特征向量 xt(h,w)x_{t}^{(h, w)}xt(h,w)​, non-local 操作本质上是从 {xk(i,j)}all\left\{\mathbf{x}_{k}^{(i, j)}\right\}_{a l l}{xk(i,j)​}all​ 中找到其相似度最大的上下文信息, 设参数为 V=[Vθ,Vϕ,Vg,Vo]\mathbf{V}=\left[\mathbf{V}_{\theta}, \mathbf{V}_{\phi}, \mathbf{V}_{g}, \mathbf{V}_{o}\right]V=[Vθ​,Vϕ​,Vg​,Vo​], 那么 non-local 可以表示为:

不难看出 SmallBig 和 non-local 本质上都是一种 attention 方式, 通过增强某些特征, 抑制另一些特征来得到更好的结果, 作者列出了 SmallBig 和 non-local 的不同:

  1. SmallBig 使用的 max pooling 来作为 hard attention, 而 non-local 是使用 similar context 作为 soft attention.
  2. attention 的感受野不一样(第一次听到这种说法), non-local 直接作用于 global spatio-temporal tube 来学习全局 relation, 可能会忽略一些比较重要的特征. 而 SmallBig 作用于 local spatio-spatial tube, 用来得到精细的表征, 再通过不断增大 big branch 的感受野来学习 local-to-global view 的表征

接下来看下实验部分:

可以看到即使不增大 big branch 中的时空感受野(仅使用 3×1×13 \times 1 \times 13×1×1 的 SmallBig block), 其效果也比不加 SmallBig block 的 3×1×13 \times 1 \times 13×1×1 3D CNN 的效果更好, 说明使用 max pooling 来得到响应值更高的上下文信息能够学到更好的特征.

可以看到为中间的 stages(e.g. res4) 添加 SmallBig block 后效果更好, 作者认为这是由于前面(后面)的 stage 的感受野太小(太大), 会产生无用(冗余)的上下文信息, 所以中间 stages 的上下文信息是更加 discriminative 的.

big branch 的感受野也不是越大越好的, 精度随着感受野的增大先上升再降低, 作者认为这是由于感受野增大上下文信息也会逐渐变得冗余导致的. 所以直接增大 3D 感受野到一个 big view 是不理智的, 需要逐渐地增大其感受野, 以 local to global 的方式更加有效:

然后是对 SmallBig block 设计的一些消融性验证, max pooling 效果比 avg pooling 好, 权重共享不但降低了参数量, 还提高了精度, 使用 Individual BN 精度更高.

Individual BN 是指 BN(conv(small))+BN(conv(big)), single BN 是指 BN(conv(small)+conv(big)).

这里的 Individual BN 提的点有点过分!
