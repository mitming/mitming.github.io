FineGym
Created2020-11-29|Updated2020-11-30|Video
|Word count:1.9k|Reading time:5min

一句话总结: 针对当前数据集不够细粒度这个问题, 本文提出了一个新的数据集, 这个数据集更加关注细粒度的行为分析, 现有的 action recognition 或者 action localization 在这个数据集上的表现都不太好, 作者也做了大量的实验, 对当前的一些方法在这个数据集上的表现做了分析和总结.

关于这篇文章的详细信息, 作者(邵典小姐姐)已经在 知乎 上写的很清楚了, 也写的非常地…生动…然后项目主页得到更加详细的数据描述, 这篇博客主要就是简单介绍下, 然后分析下实验结果.

提出了一个非常棒的数据集, 把所有的动作在语义层面按照细粒度分为了 3 个级别, 分别是:

  1. 事件类别 (event)
  2. 组类别 (set)
  3. 元素类别 (element)

在时序上分为了 动作 和 子动作, 一个动作内部会有多个子动作，这些子动作的时域位置会被标注。动作由事件类别来描述，而时域上更细粒度的子动作可以由组类别和元素类别两种粒度进行描述。

值得注意的是，FineGym中子动作的持续时间往往很短，大部分在几秒的长度，但信息量却很大（下面的实验部分有提到）。这种短时限制有效保证了数据的细粒度。

对女子的四种事件（女子跳马，女子平衡木，女子自由体操和女子高低杠）进行了“3+2”的细粒度标注。基于这四种事件类别，FineGym定义和筛选了15个组类别，并由此进一步定义了530种不同的元素类别，其中共354类目前具有子动作数据，这种情况源自于体操动作本身使用的不均衡性。FineGym保留了原始的分布，但也提供了两版不同的数据集以供选择：

  • Gym99共99类，数据分布相对更加均衡；
  • Gym288共288类，数据分布较不平衡。

好那么接下来就直接进入实验结果分析的环境, 首先看下经典方法 TSN 在整个数据集上的表现:

看起来好像还不错, 但可以看出在最粗粒度的识别(事件类别 event)中, appearance 特征的贡献远远超过了 flow 特征, 且准确率已经趋于饱和, 但在更细一点的粒度(组类别 set)上, flow 的作用就比 RGB 大了, 这说明更加细粒度的动作识别需要更加关注动作本身, 而不是一些场景/物体信息.

然后作者给出了目前主流的方法在 FineGym 上的结果:

上面的 3 个表代表 3 种不同的数据集设置

  1. elements across all events
    同时使用存在长尾问题的数据集 Gym288 和平衡的数据集 Gym99 中所有的 elements
  2. elements within an event
    从 Gym99 中选择 2 个特定事件类别(event) 的所有元素类别(element)作为数据集, 这 2 个类别分别是跳马(VT)和自由体操(FX), 分别有 4 个和 1 个 组类别 (set), 使用其他中所有的 elements
  3. elements within a set
    选择 FX-G1 这个 set 中的所有元素(element) 和 UB-G1 这个 set 中的所有元素(element) 作为数据集.

作者在这 3 种数据集下做了实验, 得到的结论如下:
1. 在第一个数据集设置中, 由于存在长尾分布, 所有的方法都会倾向于有最多数量的 element 动作, 尤其在 Gym288 数据集上
2. 相比于 RGB 信息, flow 信息更加重要, 上表中除了 VT(跳马) 这一项运动外全都是 Flow 的结果更好, 作者解释是因为跳马这个 event 中的 elements 动作都有剧烈的外观变化, 所以这项运动的 RGB 信息更加重要.
3. 对时序信息建模是很重要的, 因为 TRN 和 TSM 的效果都远远超过了 TSN
4. 以 I3D 和 NL I3D 为例, 不管是在 ImageNet 上还是在 Kinetics 上预训练, 其结果相差不大, 带 * 的是在 Kinetics 上预训练, 作者认为这是由于 FineGym 数据集的 elements 动作和 Kinetics 中动作的时序信息相差太大导致的. (因为体操中每个 element 动作都很快, 所以只有很短的几秒, 但 Kinetics 中动作的时间比这个长多了)
5. 基于骨骼信息的方法(ST-GCN)表现不佳, 作者认为原因可能是在运动项目中运动员的关键点预测的结果不好.

上面都在说不同的 action recognition 方法在这个数据集上的表现, 作者也用 SSN 来测试了 Action Localization 在数据集上的表现

可以看到对于子动作的定位显然更加困难, 也说明了 Action Localization 这个 task 其实还是有不少问题, 作者在文中提到为了更好地对 sub-action 定位, 可能需要更好地理解整个 action.

最后作者也分析了一些设置对于这个数据集有什么影响.

首先是稀疏采样得到的总帧数对结果的影响:

可以看到在 UCF101 和 Activity v1.2 这 2 个数据集上只需要几帧就能够得到更好的结果, 但在 FineGym 上则需要更多的帧, 并且准确率随着帧数稳步提升, 这表明在 FineGym 上, 每一帧都有利于细粒度的 action recognition.

然后是时序信息的影响有多大?

作者这里做了 3 个实验, 第一个实验表明 flow 能够带来比 RGB 信息更大的性能提升

第二个实验在测试时打乱了所有帧的顺序, 然后发现那些对时序信息进行建模了的方法(TSM, TRN)都出现了比较大的精度下降.

第三个实验表明, 如果测试阶段的帧数和训练阶段的帧数(动作的快慢程度)相差过大时, TSM 的精度会掉的很多, 但 TSN 却没啥影响, 因为 TSM 对时序进行了建模而 TSN 却没有.

上面三个实验都说明了对时序信息进行建模在 FineGym 这个数据集上是至关重要的.

最后作者表明, 不管模型是在 ImageNet 还是 Kinetics 上预训练, 结果都不会影响特别大, 作者认为这是由于预训练数据集和 FineGym 数据集动作之间差异过大导致的, 也就是粗粒度动作和细粒度动作在时序上存在比较大的差异导致的.

作者也分析了为什么当前的方法不能很好地在 FineGym 上刷榜…

  1. 剧烈运动
    FineGym 的运动都是比较剧烈的, 时间很短, 所以帧数很少
  2. 微妙的空间语义
    不同动作之间的语义信息差的不多, 所以不能直接通过背景或者其他东西来直接分类, 它涉及身体部位的差异，例如腿是弯曲的还是笔直的，以及人与物体之间的关系
  3. 复杂的时序动态
    例如动作的角度和旋转方向不同会导致不同的动作类别, 例如向前翻和向后翻.
  4. 关系推理
    例如萨尔托斯(某个动作, 例如旋转)的时间的不同会导致不同的动作分类
