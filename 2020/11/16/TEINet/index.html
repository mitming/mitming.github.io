TEINet:Towards an Efficient Architecture for Video Recognition
Created2020-11-16|Updated2020-12-07|Video
|Word count:842|Reading time:3min

论文: TEINet: Towards an Efficient Architecture for Video Recognition

3D CNN 参数量太大了，所以利用 2D CNN 来捕获时序信息是有必要的，虽然之前的方法 TSM 已经做到了这一点，但是 TSM 的做法是 hand-crafted 的，即手动 shift 特征来实现不同帧之间的特征融合，TIN 觉得这样做是不好的，所以提出了另一种用 2D CNN 来捕获时序信息的方式，通过把 TtimesC×H×WT times C \times H \times WTtimesC×H×W 转换为 C×T×H×WC \times T \times H \times WC×T×H×W，再用 2D channel-wise CNN 在每个通道上对时序进行卷积。

这里作者提出了 2 个模块，分别如上图左和右所示，可以理解为先用 attention 来增强和时序相关的特征，抑制和时序无关的特征，再利用一种可学习的模块给予 2D CNN 处理时序特征的能力。

  1. MEM
    MEM 模块用 2 个不同时序特征图的差来标注运动，然后用 attention 的方式来决定当前帧的哪些特征(特征图中的部分通道)能够更好地表达运动信息，即增强了 motion-related 的特征，抑制了 motion-unrelated 的特征。
  2. TIM
    TIM 模块的作用和 TSM 一样，用于赋予 2D CNN 提取时序信息的能力，和 TSM 不同的是这里并非固定地把某些特征从当前帧的特征图上 shift 到相邻帧的特征图上，而是先把特征图的 shape 从 TtimesC×H×WT times C \times H \times WTtimesC×H×W 变为 C×T×H×WC \times T \times H \times WC×T×H×W, 再使用 channel-wise 的卷积 (3×1×13 \times 1 \times 13×1×1) 在每个通道上对时序进行卷积。

作者对比了一下不同方式的参数量

  1. 对于 vanilla(原始的，没有额外改动的) 3D CNN, 其参数量为 Cout×Cin×t×d×dC_{out} \times C_{in} \times t \times d \times dCout​×Cin​×t×d×d
  2. 对于时序 1D CNN，其参数量为 Cout×Cin×tC_{out} \times C_{in} \times tCout​×Cin​×t
  3. 对于 TIM，其参数量为 Cout×1×tC_{out} \times 1 \times tCout​×1×t

这篇文章也就这么些东西，好在实验做的比较充分，一个一个来看：

  1. (a) 是为了验证 MEM 和 TIM 模块的有效性, 最原始的就是最上面的 Res50 + TSN, 在加了 MEM 后精度到了 33.5%, 如果只加 TIM 的话精度能到 46.1%, 都加则 47.4%, 可以看到 TIM 模块是比 MEM 模块更加有效的, 这表明时序上下文信息在 something-something 数据集上相比空间特征是更加重要的.
  2. (b) 验证了把 TEI 放到不同的 stage 中的效果, 可以看到放到后面几个 stage 中的效果比较好, res4res_4res4​ 比 res5res_5res5​ 多了 3 个 block, 但得到了相似的结果.
  3. © 研究了 TEI blocks 数量和精度的 trade-off, 从表中可以看出通过不断添加 TEI blocks 可以涨点, 但 res2−5res_{2-5}res2−5​ 精度只比 res4−5res_{4-5}res4−5​ 高了 0.7%, 但却多了 7 个 TEI blocks, 所以可以只在 res4−5res_{4-5}res4−5​ 内加 TEI block.
