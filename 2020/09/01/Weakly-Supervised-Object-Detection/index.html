Weakly Supervised Deep Detection Networks CVPR’16

Introduction

Problem Definition

The definition of weakly supervised object detection (WSOD) is using only image-level labels(e.g. category) to learn object detectors

Existing problems

  1. WSOD algorithms are not good enough.
  2. Existing models are not end-to-end.

Motivation

By the hypothesis that since pre-trained CNNs generalize so well tto a large number of tasks, they should contain meaningful representation of the data. Hence, CNNs trained for image classification may already contain implicitly most of the information required to perform object detection.

Contribution

Proposed the first end-to-end method for WSOD

Method

The overall idea consists of three steps:

  1. Pre-train a CNN on a large-scale classification dataset. In this paper, using ImageNet ILSVRC 2012.
  2. Construct the Weakly Supervised Deep Detection Networks(WSDNN) as an architectural modification of this CNN.
  3. Fine-tune the WSDNN on the target dataset, using only image-level annotations.

Weakly supervised deep detection network

Given the pre-trained CNN, we transform it into a WSDDN by the following three steps:

  1. Replace the last pooling layer in the last convolutional block(relu5 and pool5 for example) with a SPP layer.
    So we can get a feature vector or representation ϕ(x;R)\phi(x; R)ϕ(x;R) by a function that takes an image xxx and a region(bbox) RRR as input:

ϕ(x;R)=ϕSPP(⋅;R)∘ϕrelu5(x)\phi(\mathbf{x} ; R)=\phi_{\mathrm{SPP}}(\cdot ; R) \circ \phi_{\mathrm{relu} 5}(\mathbf{x}) ϕ(x;R)=ϕSPP​(⋅;R)∘ϕrelu5​(x)

where ϕrelu5\phi_{\mathrm{relu} 5}ϕrelu5​ needs to be computed only once for the whole image and ϕSPP(⋅;R)\phi_{\mathrm{SPP}}(\cdot ; R)ϕSPP​(⋅;R) is fast to compute for any given region RRR

  1. Given an image xxx, using Selective Search or Edge Boxes to obtain a list of candidate object regions(RoIs) R=(R1,...,Rn)R=(R_1, ..., R_n)R=(R1​,...,Rn​), then modify the SPP layer to take as input not a single region, but rather the full list RRR. In particular, ϕ(x;R)\phi(x; R)ϕ(x;R) is defined as the concatenation of ϕ(x;R1),...,ϕ(x;Rn)\phi(x; R_1), ..., \phi(x; R_n)ϕ(x;R1​),...,ϕ(x;Rn​) along the forth dimension(since each individual ϕ(x;Ri)\phi(x; R_i)ϕ(x;Ri​) is a 3D tensor).

The classification stream performs classification of the individual regions, by mapping each of regions to a C-dimensional vector of class scores to detect C different classes using a linear map ϕfc8−cls\phi_{fc8-cls}ϕfc8−cls​. Then use softmax to compute the probability of each class for individual RRR:

Detection data stream

The detection stream performs detection, by scoring regions relative to one another. This is achieved by using a linear map ϕfc8−detect\phi_{fc8-detect}ϕfc8−detect​, and also resulting in a matrix of scores: xdetect∈RC×∣R∣x^{detect} \in \mathbb{R}^{C \times |R|}xdetect∈RC×∣R∣

[σdet(xdet)]ij=exijdet∑k=1∣R∣exikdet(2)\left[\sigma_{\mathrm{det}}\left(\mathbf{x}^{det}\right)\right]_{i j}=\frac{e^{x_{i j}^{det}}}{\sum_{k=1}^{|\mathcal{R}|} e^{x_{i k}^{det}}} \tag{2}[σdet​(xdet)]ij​=∑k=1∣R∣​exikdet​exijdet​​(2)

while the two streams are remarkably similar, the introduction of the σclass\sigma_{class}σclass​ and σdet\sigma_{det}σdet​ non-linearities in the classification and detection streams is a key difference which allows to interpret them as performing classification and detection.

The softmax in classification stream compares class scores for each region, another softmax in detection stream compares the scores of different regions. Hence, the classification softmax predicts the the most likely category for a region, but the detection softmax predicts which regions are more likely contain useful information.

Combined region scores and detection

The final score of each region is obtained by takeing the element-wise product xR=σclass(xcls)⊙σdet(xdet)x^R = \sigma_{class}(x^{cls}) \odot \sigma_{det}(x^{det})xR=σclass​(xcls)⊙σdet​(xdet), the region scores are used to rank image regions, then standard NMS is performed to obtain the final list of class-specific detections in an image.

The way the two streams’ scores are combined is reminiscent of the bilinear network but there are three key differences:

  1. The different softmax operators explicitly breaks the symmetry of the two streams.
  2. Using element-wise product σclass(xcls)⊙σdet(xdet)\sigma_{class}(x^{cls}) \odot \sigma_{det}(x^{det})σclass​(xcls)⊙σdet​(xdet) rather than product of two feature vectors σclass(xcls)⊗σdet(xdet)\sigma_{class}(x^{cls}) \otimes \sigma_{det}(x^{det})σclass​(xcls)⊗σdet​(xdet).
  3. Scores(I think it is σclass(xcls)⊙σdet(xdet)\sigma_{class}(x^{cls}) \odot \sigma_{det}(x^{det})σclass​(xcls)⊙σdet​(xdet), but it’s σclass(xcls)⊗σdet(xdet)\sigma_{class}(x^{cls}) \otimes \sigma_{det}(x^{det})σclass​(xcls)⊗σdet​(xdet) in paper) are computed for specific image regions rather than a fixed set of image localizations on a grid.

Spatial Regularizer

As WSDDN is optimized for image-level class label, it does not guarantee any spatial smoothness such that if a region obtains a high score for an object class, the neighboring regions with high overlap will also have high scores.

In supervised detection case, they takes the region proposals that have at least 50% IoU with GT bbox as positive samples and then regress them into GT bbox. Since WSOD cannot use GT bbox(bbox annotations), so follow a soft regularization strategy that penalizes the feature map discrepancies at the second fully connected layer fc7fc7fc7 between the highest scoring region and the regions with at least 60% IoU during training:
